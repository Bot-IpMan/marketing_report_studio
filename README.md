# Marketing Report Studio V8 (Access + Folders, Fixed)

Локальний однофайловий застосунок для підготовки маркетингових/аналітичних звітів з режимами доступу, структурою папок та експортом клієнтської версії.

## Огляд

Проєкт складається з:

- UI-студії в одному HTML-файлі: `marketing_report_studio_v8_access_folders_fixed.html`
- прикладів/контенту звітів у Markdown по каналах/темах
- технічних нотаток щодо виправлень: `marketing_report_studio_v8_fixed_notes.md`

Версія `fixed` виправляє критичні проблеми ініціалізації JavaScript, через які попередня V8 могла не запускатися.

## Ключові можливості

- Редагування даних у режимі адміністратора:
  - додавання даних
  - авто-звіт
  - графіки
  - таблиці
  - конкурентів
  - вставка CSV
- Ролі доступу в інтерфейсі:
  - `admin` (повний доступ до редагування)
  - `viewer` (обмежений перегляд, `adminOnly` елементи приховані)
- Експорт/збереження:
  - JSON-експорт
  - збереження локальної HTML-версії
  - окрема клієнтська HTML-версія
- Папкова організація файлів звітів у sidebar.
- Вбудовані залежності в HTML (зокрема `JSZip`), без обов'язкової інсталяції пакетів.

## Швидкий старт

1. Відкрийте `marketing_report_studio_v8_access_folders_fixed.html` у браузері.
2. Працюйте у `admin`-режимі для створення/редагування звіту.
3. Використайте:
   - `Зберегти адмін` для внутрішньої версії
   - `Для клієнта` для переглядової версії
4. Передайте клієнту лише клієнтську версію HTML.

## Режими доступу

- У DOM застосовується `data-access` (`admin` або `viewer`).
- Елементи з класом `adminOnly` автоматично приховуються для `viewer`.
- Кнопка з іконкою замка (`unlock`) керує переключенням у редакторський режим.

Важливо: це UI-рівень обмеження. Воно не замінює серверну авторизацію/аутентифікацію.

## Що виправлено у fixed-версії

Згідно `marketing_report_studio_v8_fixed_notes.md`, усунені проблеми порядку ініціалізації:

- читання `reportData` без виклику ще неініціалізованого helper-а
- правильний порядок виклику `initState()` після створення DOM-посилань
- коректна робота нормалізації даних з локальним report-об'єктом
- уникнено ранніх звернень до неініціалізованого глобального `REPORT`

## Технічні примітки

- Проєкт працює як статичний HTML-документ (без backend).
- Частина даних може зберігатися в самому HTML (`<script id="reportData" type="application/json">`).
- Через вбудовані бібліотеки HTML-файл може бути великим.
- У Windows можуть з'являтися попередження про `LF/CRLF` при git-операціях.

## Обмеження

- Немає справжньої системи логіну/прав доступу на рівні сервера.
- Захист клієнтського контенту залежить від того, який саме файл ви передаєте.
- Для командної роботи без конфліктів бажано домовитися про єдиний процес збереження/мерджу HTML.

## Рекомендації для розвитку

- Винести бізнес-логіку з одного HTML у модулі (`/src`).
- Додати автотести на нормалізацію даних і рендер ключових віджетів.
- Додати schema-validation для `reportData`.
- Перейти на серверну модель авторизації, якщо потрібен реальний контроль доступу.

## Ліцензії та залежності

- У проєкті використовується `JSZip` (див. коментарі всередині HTML щодо ліцензування).
- Для сторонніх даних/текстів у папках звітів перевіряйте права використання окремо.
